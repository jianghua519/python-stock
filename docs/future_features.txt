# 📄 未来功能子项目建议清单

---

【分类一】采集扩展（Data Ingestion）

1. 支持更多数据源（高优先级｜中难度）
   - 除了现有网站/API，再增加如TSE官网、Yahoo Finance、QUICK等渠道
   - 目标：数据源多样化，防止单点依赖

2. 实时流数据采集（中优先级｜高难度）
   - 通过WebSocket、实时API订阅股票成交推送
   - 目标：从日内多次定时抓取→实时流处理

3. 补数据机制（中优先级｜中难度）
   - 若某天数据抓取失败，支持自动补抓历史数据
   - 目标：提升数据完整性和稳定性

---

【分类二】分析扩展（Data Analysis）

4. 引入更多技术指标（高优先级｜低难度）
   - 例如MACD、RSI、布林带（Bollinger Band）、成交量变化率
   - 目标：丰富分析维度，提升策略多样性

5. 多周期分析（中优先级｜中难度）
   - 支持不同时间粒度（分钟、小时、日、周）
   - 目标：提供短线、中线、长线多种分析策略支持

---

【分类三】AI智能扩展（AI Model）

6. 多模型融合预测（高优先级｜高难度）
   - LSTM、Transformer、Random Forest等集成
   - 目标：提升预测准确率和模型稳健性

7. AI自动再训练（中优先级｜高难度）
   - 定期根据最新数据自动再训练和更新模型
   - 目标：保证模型随市场变化持续优化

8. AI推理服务负载均衡（低优先级｜高难度）
   - 多模型实例化+负载均衡策略
   - 目标：提升大批量推理响应性能

---

【分类四】系统运维扩展（Infrastructure）

9. Prometheus+Grafana监控（高优先级｜中难度）
   - 服务健康检查、响应时间、错误率监控
   - 目标：系统可靠性可视化、自动报警

10. ELK日志中心（中优先级｜中难度）
    - ElasticSearch收集日志，Kibana可视化
    - 目标：快速定位生产环境异常

11. 自动化部署（中优先级｜中难度）
    - GitHub Actions+Docker Registry+Production Server
    - 目标：每次代码合并后自动部署到生产环境

---

【分类五】API与前端扩展（API & Frontend）

12. API文档自动生成（高优先级｜低难度）
    - FastAPI自带Swagger / Redoc界面
    - 目标：方便接口测试与对接第三方应用

13. 前端图形化展示（中优先级｜中难度）
    - 基于React或Vue绘制趋势图、分析图
    - 目标：提升可视化交互体验

14. 提供个性化推荐API（低优先级｜高难度）
    - 根据用户关注历史和操作偏好推荐股票
    - 目标：构建智能投顾雏形

---

# 🚀 推荐开发顺序（简洁版）

1. 增加数据源 → 增加指标 → API自动文档化  
2. 上监控（Prometheus）→ 加日志中心（ELK）  
3. 多模型融合 → 自动再训练 → 实时采集流化  
4. 引入前端可视化 → 提供个性化推荐服务  
5. 最后完善自动部署、负载均衡、高并发优化
